## 代码审查发现（按严重级别）

### 高风险

1. Agent 模式默认值与描述不一致，会导致 schema 替换默认不生效

- 现象：命令参数描述写的是 data layer 默认 supabase，但实现只有当显式传入 data-layer=supabase 才会注入 schema。
- 影响：在 agent 模式下，如果用户不传 data-layer，生成项目中的 SQL 仍保留 **SCHEMA**，与预期默认 public 不一致。
- 证据：
  - 参数描述为默认 supabase：[packages/cli/src/commands/init.ts](packages/cli/src/commands/init.ts#L35)
  - 实际传递 dataLayer 为可选值，无默认化：[packages/cli/src/commands/init.ts](packages/cli/src/commands/init.ts#L57)
  - 仅在 dataLayer === supabase 时注入 schema：[packages/cli/src/adapters/agent.ts](packages/cli/src/adapters/agent.ts#L85)

### 中风险

2. Auth 页面仍有手动 locale 拼接残留，和 i18n Link 自动前缀机制不一致

- 现象：signup/login/forgot/reset 中仍存在 href=/${locale} 或 href=/${locale}/terms。
- 影响：在 next-intl createNavigation 下，手动拼接 locale 可能引发重复前缀或路由不一致，属于潜在回归点。
- 证据：
  - i18n Link 来源于 createNavigation（自动 locale）：[templates/web-nextjs/apps/web/src/i18n/navigation.ts](templates/web-nextjs/apps/web/src/i18n/navigation.ts#L4)
  - 残留手动拼接示例：[templates/web-nextjs/apps/web/src/app/[locale]/(auth)/signup/page.tsx](<templates/web-nextjs/apps/web/src/app/[locale]/(auth)/signup/page.tsx#L46>)
  - 残留 terms 手动拼接：[templates/web-nextjs/apps/web/src/app/[locale]/(auth)/signup/page.tsx](<templates/web-nextjs/apps/web/src/app/[locale]/(auth)/signup/page.tsx#L155>)
  - login 中同类写法：[templates/web-nextjs/apps/web/src/app/[locale]/(auth)/login/page.tsx](<templates/web-nextjs/apps/web/src/app/[locale]/(auth)/login/page.tsx#L56>)

### 低风险

3. 命名细节可读性问题

- 现象：replaceSchemePlaceholder 命名应为 replaceSchemaPlaceholder（拼写语义不一致）。
- 影响：不影响运行，但影响长期可维护性与代码沟通成本。
- 证据：[packages/cli/src/core/scaffold.ts](packages/cli/src/core/scaffold.ts#L123)

## 预评估结论（小样本）

结论：在本次真实改动样本中，deepseek v4 pro 的编码能力达到 可用到中上 水平，适合在有明确 OpenSpec 任务约束下承担功能实现；但在默认值一致性、路由机制细节对齐方面仍需要稳定的人类 review 闸门。

建议评分（仅限本样本，不外推）：

- 需求实现完整度：8.0/10
- 代码工程化质量：7.5/10
- 细节一致性与回归风险控制：6.5/10
- 综合建议：7.3/10（可用，需 review + 验收）

## 实施建议

1. 保留当前工作流

- 继续采用 Claude 规划 + deepseek 实现 + 人类/高可靠模型 review 的三段式。

2. 为该类改动补两类回归检查

- Agent 模式默认参数回归（未传 data-layer 时 schema 应按默认策略生效）
- i18n Link 路由回归（禁止手动 locale 拼接）

3. 在任务模板中新增一致性检查项

- 参数描述中的默认值必须与运行时默认逻辑一致。
- 使用 i18n Link 时禁止 href 手工拼接 /${locale}。
