## 证据边界与写作原则

为确保“洞察、观点、论点”实事求是，本章节采用如下规则：

- 事实性陈述必须能回溯到公开来源（仓库 README、官方文档或源码说明）。
- 推导性结论会明确标记为“迁移建议/工程推导”，不伪装为上游项目已声明事实。
- 涉及开源项目时均附出处链接，避免无源论断。

## 本仓库现状基线

与当前仓库状态一致：

- 本仓库当前不包含独立的 AgentDock CLI 可执行包实现。
- 当前内容以架构规划、方法论借鉴和落地路线为主。
- 因此本章节属于“面向实施的设计文档”，不是“已交付功能手册”。
