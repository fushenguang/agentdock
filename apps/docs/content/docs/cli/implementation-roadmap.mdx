# 落地路线

## 当前状态（2026-06-04）

CLI `@cogito.ai/cli` 已正式发布并可用于生产。

已完成阶段：Phase 0 · Phase 1 · Phase 2 · Phase 3。

---

## Phase 0 ✅ 先定协议，不先堆功能

**目标**：形成机器可消费且可测试的契约基线。

**已交付**：
- 命令输入 schema（Citty flag 定义，含类型与默认值）
- 标准输出 schema（`{ ok: true, ... }` / `{ ok: false, error, message }`）
- 错误码映射：`MISSING_ARG` / `TEMPLATE_NOT_FOUND` / `TARGET_DIR_EXISTS` / `CLI_VERSION_OUTDATED` / `SCAFFOLD_FAILED`

**验收结果**：同一输入在本地与 CI 产出一致结构 ✓，错误分型稳定 ✓

---

## Phase 1 ✅ 最小可用 CLI（无头优先）

**目标**：先跑通最短链路。

**已交付**：
- `agentdock init` 无头模式：`--name`、`--template`、`--pm`、`--dir`、`--json`、`--silent`
- NDJSON 输出：成功/失败均输出单行 JSON 对象到 stdout
- 失败路径返回结构化错误 + 非零退出码

**验收结果**：机器模式 stdout 零噪音 ✓，失败路径结构化 ✓

---

## Phase 2 ✅ 人类交互与网络鲁棒性

**目标**：在不破坏无头契约的前提下提升可用性。

**已交付**：
- TTY 交互流程：Clack 驱动的项目名、模板选择、包管理器选择、确认步骤
- 模板内置于 CLI npm 包（无网络依赖，离线可用）

**验收结果**：Human/Machine 模式共享同一执行器核心 ✓

---

## Phase 3 ✅ Agent 投影（MCP）

**目标**：把 CLI 能力暴露为 AI Agent 可直接调用的工具面。

**已交付**：
- `agentdock mcp`：MCP Stdio 服务器
- `list_templates` 工具：列出所有可用模板
- `scaffold_project` 工具：脚手架项目，支持 name / template / targetDir / packageManager 参数

**验收结果**：工具参数可由 schema 约束 ✓，调用失败返回结构化错误 ✓

---

## Phase 4 — 高阶能力扩展（规划中）

**目标**：在稳定协议上扩展价值密度。

**待交付**：
- `agentdock add` — 向现有项目中增量添加能力（如 auth、i18n）
- `agentdock doctor` — 检查项目配置健康度
- `agentdock migrate` — 跨版本迁移辅助
- 远程模板注册表（Giget 接入，支持私有源与镜像）
- 模板能力与知识资产（SKILL.md、schema、recipe）协同版本发布

**验收标准**：
- 高阶命令不破坏既有 JSON 协议兼容性
- 文档、skills、行为三者版本一致

---
