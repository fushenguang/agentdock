# UI Design in the AI Era

## Why This Problem Matters

In the AI Coding Era, coding ability has been democratized — but design ability has not. When you ask an AI to "build a beautiful landing page," what you get is AI Slop: Inter font + purple-blue gradients + nested rounded cards + glowing dark mode text. It looks polished but has no soul, no brand identity, and no differentiation.

The root cause is simple: **without a design system constraint, AI can only produce what it saw most frequently in training data.**

## What This Topic Solves

This topic provides a systematic approach to making AI Coding Agents produce **distinctive, high-quality, maintainable UI**:

1. **Design System** — A token-based CSS system (OKLCH + Tailwind v4 `@theme`) that AI can understand and follow
2. **Workflow** — A human-AI collaborative workflow (Impeccable) that ensures quality at every step
3. **Anti-Patterns** — A catalog of AI Slop patterns with automated detection
4. **Research Framework** — A methodology for finding design inspiration and best practices

## The Human + Machine + AI Agent Trinity

A new interaction paradigm for design in the AI era:

- **Human**: Sets direction, makes aesthetic judgments, approves or rejects AI proposals
- **AI Agent**: Generates and implements based on design system constraints
- **Machine (CI/CLI)**: Runs deterministic quality checks (Impeccable `detect`) to catch AI Slop before it reaches production

Run the following in your project root to initialize the design system:

```bash
npx impeccable init
```

Or manually create `DESIGN.md` following the [CSS System](/en/docs/ui-design/css-system) principles.
